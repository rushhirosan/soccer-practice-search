## 秘密情報の取り扱い

### 環境変数ファイル
以下のファイルは**絶対にコミットしないでください**：

- `.env`
- `.env.local`
- `.env*.bak`
- `.env*.backup`
- その他の環境変数を含むファイル

### .gitignoreの確認
コミット前に`.gitignore`を確認し、秘密情報を含むファイルが除外されていることを確認してください。

### 漏洩時の対応
秘密情報が漏洩した場合：

1. **即座に無効化**
   - APIキー、トークンなどを無効化
   - 新しいキーを生成

2. **Git履歴から削除**
   ```bash
   git filter-branch --force --index-filter \
     'git rm --cached --ignore-unmatch <ファイル名>' \
     --prune-empty --tag-name-filter cat -- --all
   ```

3. **強制プッシュ**（注意が必要）
   ```bash
   git push origin --force --all
   ```

4. **GitHub Secret Scanning**
   - GitHubの「Security」タブでアラートを確認
   - アラートを解決済みとしてマーク

## セキュリティベストプラクティス

1. **環境変数の使用**
   - 秘密情報は環境変数として管理
   - 本番環境ではFly.ioのシークレット機能を使用

2. **定期的な監査**
   - 定期的にリポジトリをスキャン
   - GitHub Secret Scanningを有効化

3. **最小権限の原則**
   - APIキーには必要最小限の権限を付与
   - 定期的に権限を見直し

4. **ログ管理**
   - ログに秘密情報を出力しない
   - エラーメッセージに機密情報を含めない
